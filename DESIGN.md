# 이나네 가족자산 디자인 시스템

## 제품 인상

가족의 중요한 재무 정보를 다루므로 **차분함, 정확함, 신뢰감**을 우선한다. 장식보다 숫자의 의미와 데이터 상태가 먼저 보여야 하며, AI는 별도 세계관이 아니라 사용자의 다음 행동을 돕는 보조자처럼 표현한다.

## 핵심 원칙

1. 숫자를 과장하지 않는다. 부채나 환율처럼 빠진 데이터가 있으면 범위를 함께 표시한다.
2. 색상만으로 의미를 전달하지 않는다. 부호, 아이콘, 텍스트를 함께 사용한다.
3. 가족 합산이 기본이며 필요할 때 남편·아내별로 구분한다.
4. 입력보다 확인이 중요하다. 저장 결과와 오류 원인을 즉시 보여준다.
5. 모바일에서 자주 쓰는 가계부와 계좌 흐름을 최우선으로 최적화한다.

## 색상

### 기본

- Canvas: `#f8fafc` — 전체 화면 배경
- Surface: `#ffffff` — 카드와 표
- Ink: `#0f172a` — 주요 제목과 숫자
- Body: `#334155` — 본문
- Muted: `#64748b` — 보조 정보
- Hairline: `#e2e8f0` — 구분선
- Primary: `#1d4ed8` — 주요 행동과 선택 상태

### 금융 의미

한국 금융 서비스 관례를 따른다.

- 수익·상승: 빨강 `#dc2626`
- 손실·하락: 파랑 `#2563eb`
- 수입·완료: 초록 `#15803d`
- 경고·확인 필요: 주황 `#d97706`
- 오류·삭제: 진한 빨강 `#b91c1c`

수익과 손실에는 반드시 `+/-`, 상승·하락 아이콘 또는 설명을 같이 사용한다.

## 타이포그래피

- 한글 우선 시스템 스택: Pretendard → Apple SD Gothic Neo → Noto Sans KR → Segoe UI
- 페이지 제목: 24px / 700
- 핵심 자산 숫자: 30–32px / 700, tabular numerals 권장
- 카드 제목: 14px / 600
- 본문: 14px / 400
- 보조 정보: 12px / 400

외부 폰트 다운로드에 앱 실행과 빌드가 의존하지 않게 한다.

## 핵심 화면

### 대시보드

위에서 아래 순서로 배치한다.

1. 등록 자산 합계와 데이터 기준일
2. 이번 달 수입·실지출·순저축
3. 확인이 필요한 데이터와 이상징후
4. 포트폴리오·부동산 구성
5. AI 제안과 다음 행동

부채가 구현되기 전까지 `순자산`이라는 표현을 사용하지 않는다.

### 가계부

`요약 / 거래내역 / 이체 / 예산`으로 정보 구조를 분리한다. 내부이체는 실지출에 포함하지 않고, 외부이체는 지출로 포함한다. 현재 선택된 월과 결제수단 필터는 항상 보여준다.

### 포트폴리오

평가금액, 수익금, 수익률, 비중을 우선한다. 외화 자산은 통화와 환율 기준을 표시한다. 모바일 표에서는 종목명·평가금액·수익률을 우선하고 나머지는 상세 보기로 이동한다.

### 빈 화면과 오류

- 빈 화면: 이유 + 첫 행동 버튼
- 로딩: 화면 구조를 유지하는 스켈레톤
- 오류: 원인 요약 + 재시도
- 저장 성공: 변경된 대상이 명확한 토스트

숫자 조회 실패를 `0원`으로 대체하지 않는다.

## 접근성

- 아이콘 전용 버튼에 `aria-label` 제공
- 포커스 표시를 제거하지 않음
- 폼 오류를 입력 필드와 연결
- 그래프 데이터는 표 또는 텍스트 요약도 제공
- 명도 대비 WCAG AA 이상 유지

## 금지 사항

- 금융 의미 색상을 화면마다 다르게 사용하지 않는다.
- 데이터가 없거나 실패한 상태를 0으로 표시하지 않는다.
- 브라우저 `alert`와 `confirm`을 장기 UI로 사용하지 않는다.
- 하나의 화면에 입력·설정·분석·내역을 모두 쌓지 않는다.
- AI 결과를 확정적 금융 조언처럼 표현하지 않는다.
